# Examples

These examples are not framework-specific templates. They are small, shareable case studies that show how Forge turns a raw request into staged artifacts.

Each example includes:

- `request.md` - the original user request
- `prompt.md` - the strengthened brief produced in stage 1
- `research.md` - repository-aware findings from stage 2
- `plan.md` - the implementation plan produced in stage 3
- `review.md` - the final review summary from stage 5
- `result.md` - a short summary you can reuse in docs or launch posts

Use them to:

- show first-time users what a completed workflow looks like
- demonstrate that Forge fits feature work, refactors, and bugfixes
- link concrete examples in release notes and community submissions
